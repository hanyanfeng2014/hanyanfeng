Software Used软件使用

This guide has been tested with:本指南已经测试:

  • openDCIM version 2.1, the current version as of the writing of the guide, but it should apply to previous versions, and possibly to future ones as well.openDCIM 2.1版本,当前版本的写作指导,但它应该适用于以前的版本,可能未来的。
  • CentOS 6.4 Minimal 64bit, with the latest updates as of 2013-05-25, but it should apply to previous versions, and possibly to future ones as well.CentOS 6.4最小的64位,用最新的更新的2013-05-25,但它应该适用于以前的版本,可能未来的。
  • The的 vimeditor is used in the commands below, because it adds nice colored syntax highlighting when editing configuration files. 编辑器中使用下面的命令,因为它增加了漂亮的彩色语法高亮显示在编辑配置文件。If you don't have如果你没有 vim, you can use,你可以使用 vi, which comes preinstalled with CentOS.,与CentOS预装。

Install Apache, PHP, MySQL安装Apache、PHP、MySQL

Install, start and enable Apache:安装,启动,使Apache:

 yum -y install httpd
 service httpd start
 chkconfig httpd on

Install PHP, and the MBSTRING module required for internationalization:安装PHP,国际化所需的MBSTRING模块:

 yum -y install php
 yum -y install php-mysql
 yum -y install php-mbstring

Install, start and enable MySQL Server:安装、启动和支持MySQL服务器:

 yum -y install mysql-server
 service mysqld start
 chkconfig mysqld on

Secure MySQL Server:安全的MySQL服务器:

 mysql_secure_installation

During this step, you will:在这个步骤中,您将:

  1. Set a root password设置一个根密码
  2. Remove anonymous users删除匿名用户
  3. Disallow root login remotely不允许远程root登录
  4. Remove test database and access to it删除测试数据库和访问
  5. Reload privilege tables重新加载权限表

Create a database for openDCIM (change the database name创建一个数据库openDCIM(改变数据库名称 dcimand the password和密码 dcimpasswordto something that suits you):这适合你):

 mysql -u root -p
 mysql> create database dcim;
 mysql> grant all privileges on dcim.* to 'dcim' identified by 'dcimpassword';

Enable HTTPS启用HTTPS

Install Apache SSL Module:安装Apache SSL模块:

 yum -y install mod_ssl

Generate the necessary keys and copy them to the proper directories:生成所需的密钥,并将其复制到适当的目录:

 cd /root
 openssl genrsa -out ca.key 1024 
 openssl req -new -key ca.key -out ca.csr
 openssl x509 -req -days 365 -in ca.csr -signkey ca.key -out ca.crt
 cp ca.crt /etc/pki/tls/certs
 cp ca.key /etc/pki/tls/private/ca.key
 cp ca.csr /etc/pki/tls/private/ca.csr

To let Apache know of the proper locations of the keys, edit让Apache知道合适的位置的关键,编辑 /etc/httpd/conf/httpd.conf...

 vim +/SSLCertificateFile /etc/httpd/conf.d/ssl.conf

... …comment out the line...注释掉的行…

 SSLCertificateFile /etc/pki/tls/certs/localhost.crt

... …and substitute with:和替代:

 SSLCertificateFile /etc/pki/tls/certs/ca.crt

Also comment out the line...也注释掉的行…

 SSLCertificateKeyFile /etc/pki/tls/private/localhost.key

... …and substitute with:和替代:

 SSLCertificateKeyFile /etc/pki/tls/private/ca.key

To set the server name:设置服务器名称:

 vim +/ServerName /etc/httpd/conf/httpd.conf

Find the line...找到线……

 #ServerName www.example.com:80

...…and add below it:并添加以下:

 ServerName opendcim.example.com:443

Finally restart Apache...最后重新启动Apache……

  service httpd restart

Create a VirtualHost创建一个虚拟主机

 vim +/NameVirtualHost /etc/httpd/conf/httpd.conf

Find the line:找到:

 #NameVirtualHost *:80

... …and add below it:并添加以下:

 NameVirtualHost *:443

Now create a new configuration file for the openDCIM VirtualHost...现在创建一个新的配置文件openDCIM虚拟主机…

 vim /etc/httpd/conf.d/opendcim.example.com.conf

... …add the lines...添加行……

 <VirtualHost *:443>
     ServerAdmin marios@zindilis.com
     DocumentRoot /var/www/opendcim
     ServerName opendcim.zindilis.net
     <Directory /var/www/opendcim>
         AuthType Basic
         AuthName "openDCIM"   
         AuthUserFile /var/www/.htpasswd
         Require valid-user
     </Directory>
 </VirtualHost>

Enable User Authentication启用用户身份验证

You have protected the openDCIM web directory with a requirement for Basic authentication, with the lines already added in your Apache configuration file above.你有保护openDCIM web目录要求基本身份验证,与上面的线已经添加在您的Apache配置文件。

Now, to create at least on user, do:现在,创建至少在用户,做:

 touch /var/www/.htpasswd
 htpasswd /var/www/.htpasswd Administrator

You will be asked for a password for user "Administrator" twice.您将被要求输入密码为用户“管理员”两次。

Open Web Access on Firewall开放网络访问防火墙

The的 iptablesfirewall is enabled on CentOS be default, and blocks access to HTTPS port 443. 启用了防火墙在CentOS是默认,块访问HTTPS端口443。To allow it...让它……

 vim /etc/sysconfing/iptables

... …find the line...找到线……

 -A INPUT -m state --state NEW -m tcp -p tcp --dport 22 -j ACCEPT

...…and add below it:并添加以下:

 -A INPUT -m state --state NEW -m tcp -p tcp --dport 443 -j ACCEPT

Restart iptables:重启iptables:

 service iptables restart

Download and Install openDCIM下载并安装openDCIM

Download the latest version of openDCIM from下载最新版本的openDCIMopendcim.comopendcim.com(version 3.2 at the time of updating this guide):(版本3.2的时候更新本指南):

 cd /var/www
 curl -O http://www.opendcim.org/packages/openDCIM-3.2.tar.gz

Extract the archive and create a symbolic link:提取归档和创建一个符号链接:

 tar zxvf openDCIM-3.2.tar.gz
 ln -s openDCIM-3.2.1 opendcim

The symbolic link is not required. 符号链接并不是必需的。If you don't want to create it, just rename the directory如果您不想创建,重命名目录 openDCIM-2.1to来 opendcim. 。However, having a symbolic link in place allows you to find out the version of openDCIM at a glance, and makes upgrades easier.然而,在一个符号链接可以让你找到的版本openDCIM乍一看,并且使升级容易。

Now, prepare the configuration file for access to the database:现在,准备访问数据库的配置文件:

 cd /var/www/opendcim
 cp db.inc.php-dist db.inc.php
 vim db.inc.php

Edit the following lines, to reflect your settings of database host (in this example编辑以下行,以反映您的设置的数据库主机(在这个例子中 localhost), database name ()、数据库名(dcim), and credentials that you assigned when creating the database:),和凭证,您在创建数据库时分配:

 $dbhost = 'localhost';
 $dbname = 'dcim';
 $dbuser = 'dcim';
 $dbpass = 'dcimpassword';

Finally, restart Apache one last time:最后,最后一次重新启动Apache:

 service httpd restart

Now, you can open openDCIM in your browser to proceed with the installation. 现在,您可以在您的浏览器中打开openDCIM继续安装。If you have set up a DNS entry for a domain name to point to your web server's IP, you will go to something similar to如果你有设置域名的DNS条目指向您的web服务器的IP,你将去类似https://opendcim.example.com/https://opendcim.example.com/. 。Otherwise, you can visit否则,你可以访问https://IP_of_web_server/https://IP_of_web_server/.

In any case, you will be asked to accept the web server's HTTPS certificate. 在任何情况下,你将被要求接受HTTPS web服务器的证书。This will only happen once per browser. 这只会发生一次浏览器。You will then be asked for the Authentication credentials that you configured earlier with你将被要求您先前配置的身份验证凭证 htpasswd.

To finish with the installation, follow安装完成,跟进Scott Milliken's (lead developer) video tutorial on YouTube斯科特·米利肯(首席开发人员)在YouTube上的视频教程, starting from minute 09:00. ,从上午9点一刻。He will walk you through the first settings to get you started in the web interface.他将指导您完成第一个设置让您开始在web界面。
